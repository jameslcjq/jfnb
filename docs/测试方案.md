## 三、待补测试(缺口,按优先级)

### P0(发布前必须)

**T1 公办幼儿园无报表草稿(71 所,真实数据)**
包内有 71 所独立填报的公办幼儿园,草稿路径从未用真实公办园数据测过(现有彩排只筛民办 lsgxdm=25)。
做法:扩展 `batch-draft-from-package.js` 增加筛选 `lsgxdm∈{11,21} && xxlbdm=8`,其余流程复用。
通过标准:全部强制=0;公办园特有规则(322007 生均、338 保育费、12436 等)提示有说明。

**T2 多学段民办拆分记录(14 所:九年制9/完中4/十二年制1)**
这些学校平台要求拆分填报(ZXXCFMode=2),但 `generatePrivateDraft` 未接 `splitComputedByStage`,只产全校合计。
做法:①草稿路径接入拆分(与正式路径同款);②彩排脚本核验各学段文件强制=0、加总=全校(参照外国语实验的正式路径验法)。
通过标准:14 所各产 N+1 个文件,学段记录强制=0,逐格加总=合计。

**T3 v2.0.4 打包 + 安装冒烟**
2.0.3 安装包不含 `b608aa3` 民办修复。升版打包后在干净 Win10/11 机器:
- 安装 → 首启数据目录生成《校验规则》4 件(系统公式/自定义公式/学校属性/说明库);
- 生成 1 所公办(五件套拖入)+ 1 所民办草稿,预览显示校验面板与说明,输出目录有报表+《校验情况说明.txt》;
- 免费版限制仍生效(隐藏支出表、禁导出),激活码解锁正常。

### P1(尽快)

**T4 UI 手工用例**(见附录 A):首次设置确认流、文件检测一键生成、经费年报预览(校验面板/说明/变动建议/学段文件)、民办表单按学段显隐、设置中心、手工修正后重新校验。

**T5 边界场景**(构造数据,加入单测或彩排):
- 新建校:无上年经费年报 → 草稿应给出明确报错/引导,不崩溃;
- 学生数为 0 / 教职工为 0 的园;
- 有债务的学校(债务表非零,j2_5 规则 34 条实测——16+37 校样本全为零债务);
- 上年报表行错位/旧版模板(splitByPreviousRows 按行号读的兼容性);
- 民办有捐赠/举办者抽回/利息组合(单测已有,补真实包内有值学校)。

### P2(条件允许)

**T6 性能**:296 所全量重建+校验一轮的耗时与内存(现 37 所约十几秒,预计可接受);
**T7 年度换届演练**:模拟明年流程——新公式 xlsx 替换、`build-school-attributes.js`、`build-explanation-library.js` 重跑、说明库/属性热替换(数据目录覆盖)后生成不需发版。
