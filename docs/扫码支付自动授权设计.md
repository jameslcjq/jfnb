# 扫码缴费自动开通授权 — 设计方案

状态:设计稿(2026-07-13)。支付宝当面付先行,微信 Native 支付第二期。
前提:已有营业执照;授权服务器 jyj.yunbg.vip 已具备签发/校验/席位/停用能力。

## 1. 总体流程

```
桌面端(授权中心页)               授权服务器(jyj.yunbg.vip)            支付宝/微信
  │ 1. POST /api/pay/v1/orders ───► 创建订单(价格取服务端配置,不信客户端)
  │                                  2. 调 当面付precreate / Native下单 ──►
  │                                  ◄── qr_code 字符串 ────────────────
  │ ◄── {order_no, qr_image, amount}
  │ 3. 展示二维码+倒计时,每3秒轮询
  │    GET /api/pay/v1/orders/:no ─►
  │                                  ◄── 4. 用户手机扫码付款 ──── 用户
  │                                  ◄── 5. 异步回调 notify(验签+幂等) ──
  │                                  6. 落账 → 自动签发/续期授权
  │ 7. 轮询得到 paid + license_key
  │ 8. 自动保存激活码 → verify → 支出表解锁、导出恢复
```

## 2. 服务端设计(授权服务器内新增 pay 模块)

### 2.1 数据表

```sql
-- 订单表
CREATE TABLE pay_orders (
  id INTEGER PRIMARY KEY,
  order_no TEXT UNIQUE NOT NULL,        -- 商户订单号:yyyymmdd + 随机,不可猜
  product_key TEXT NOT NULL,            -- fund-annual-report
  plan TEXT NOT NULL,                   -- yearly
  channel TEXT NOT NULL,                -- alipay | wechat
  amount_fen INTEGER NOT NULL,          -- 金额(分),来自服务端价格配置
  school_name TEXT NOT NULL,            -- 购买时填写,= 授权 customer_name
  device_id TEXT DEFAULT '',            -- 发起购买的机器码(排查用)
  renew_license_key TEXT DEFAULT '',    -- 非空 = 续费单
  contact TEXT DEFAULT '',              -- 可选联系方式(开发票用)
  status TEXT NOT NULL DEFAULT 'pending',  -- pending|paid|expired|refunded
  provider_trade_no TEXT DEFAULT '',    -- 渠道流水号
  license_key TEXT DEFAULT '',          -- 签发/续期后回填
  notify_raw TEXT DEFAULT '',           -- 回调原文,审计
  created_at TEXT NOT NULL,
  paid_at TEXT DEFAULT ''
);

-- 价格配置(表或配置文件均可,服务端说了算)
-- product_key + plan → { title, amount_fen, years }
-- 例:fund-annual-report / yearly → { "经费年报完整版·一年", 29900, 1 }
```

### 2.2 API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/pay/v1/products?product_key=` | 返回价格/名称,桌面端展示 |
| POST | `/api/pay/v1/orders` | 创建订单并取二维码 |
| GET | `/api/pay/v1/orders/:order_no` | 轮询状态;paid 时带 license_key |
| POST | `/pay/notify/alipay` | 支付宝异步回调(form,RSA2 验签,返回 "success") |
| POST | `/pay/notify/wechat` | 微信 APIv3 回调(验签+AES-GCM 解密,返回 {"code":"SUCCESS"}) |

**POST /api/pay/v1/orders 请求**:
```json
{ "product_key": "fund-annual-report", "plan": "yearly",
  "channel": "alipay", "school_name": "XX县XX小学",
  "device_id": "...", "renew_license_key": "", "contact": "" }
```
**响应**:
```json
{ "ok": true, "order_no": "20260713xxxxxx", "amount_fen": 29900,
  "qr_image": "data:image/png;base64,...", "expires_in": 1800 }
```
二维码由服务端用 `qrcode` 库渲染成 base64 PNG,桌面端只 `<img>`,不新增渲染端依赖。

### 2.3 支付渠道抽象

```
providers/alipay-f2f.js   → precreate(order) / verifyNotify(body) / queryOrder(order_no)
providers/wechat-native.js → 同上三个方法
```
上层订单逻辑只面向这三个方法,支付宝先实现,微信后补一个文件即可。

### 2.4 签发逻辑(回调成功后)

- **新购**(renew_license_key 为空):
  创建授权:`customer_name = school_name`,`plan = 'yearly'`,
  `expires_at = 今天 + years`, `seats = 2`(留换机余量),
  **`features = { role:'school', deployment_mode:'standalone' }`** ← 必须,否则激活后被误判为联网版。
- **续费**:校验旧 key 存在 → `expires_at = max(now, 原到期日) + years`,不发新码。
- 幂等:状态机 pending→paid 单向;同 order_no 重复回调直接返回成功不重复签发。

### 2.5 安全与兜底

- 金额只认服务端配置;回调必须验签;notify 原文落库。
- 二维码 30 分钟过期(`timeout_express=30m`);过期订单标 expired。
- **回调丢失兜底**:轮询接口发现订单 pending 且已创建 >30 秒时,主动向渠道查单
  (alipay.trade.query / 微信 transactions/out-trade-no),查到已付即按回调同样逻辑落账。
- 每日对账任务:扫 24h 内 pending 单批量查渠道。
- 退款:不做自动退款;管理后台人工标 refunded + 停用对应 license。
- 对公转账/批量采购:保留现有人工发码通道,扫码不是唯一入口。

## 3. 桌面端设计(授权中心页新增"购买/续费"区)

- 无有效授权:显示 `购买完整版 ¥XXX/年`(价格从 /products 拉,改价不用重新打包)。
- 有授权(有效或过期):按钮变 `续费一年`,自动带上现有 key。
- 学校名称输入框:预填单机设置的 unitName(即授权 customer_name),可改。
- 通道:支付宝 / 微信 两个按钮(微信未开通前只显示支付宝)。
- 点击 → 弹出二维码 + 倒计时 + 金额;每 3 秒轮询;
  成功 → 自动 `licenseSaveKey` + `verify` → 界面变"已授权",支出表解锁,
  并把激活码明文展示一次提示用户留存(备重装)。
- 辅助入口:"已付款未到账?点此刷新"(触发服务端主动查单);
  "需要发票/对公转账请联系:XXX"。

## 4. 去支付宝/微信申请时要拿到的东西

### 支付宝(open.alipay.com,用企业/个体户支付宝账号)
1. 创建"网页/移动应用" → 得 **APPID**
2. 给该应用**签约"当面付"**产品(费率 0.38%)
3. 用官方密钥工具生成 RSA2 密钥对,上传应用公钥 → 得 **支付宝公钥**
4. 需要保存三样:**APPID、应用私钥、支付宝公钥**
5. notify_url 代码里传,无需预配置

### 微信支付(pay.weixin.qq.com,需营业执照)
1. 申请**微信支付商户号(mchid)**
2. 坑:Native 支付必须绑定一个 **AppID** —— 需要"已认证服务号"(300元/年)
   或微信开放平台"网站应用"(要求网站已 ICP 备案;jyj.yunbg.vip 若已备案可用)
3. 开通 **Native 支付**产品权限(费率 0.6%)
4. 需要保存:**mchid、APIv3 密钥(自设32位)、商户API证书私钥 apiclient_key.pem + 证书序列号、绑定的 AppID**

### 服务器上要配的环境变量(实现时用)
```
ALIPAY_APP_ID / ALIPAY_APP_PRIVATE_KEY / ALIPAY_PUBLIC_KEY
WXPAY_MCHID / WXPAY_APIV3_KEY / WXPAY_CERT_SERIAL / WXPAY_PRIVATE_KEY / WXPAY_APPID
PAY_NOTIFY_BASE=https://jyj.yunbg.vip
```
