# 修改说明

本次修改目标：在不大改业务逻辑、不影响现有使用习惯的前提下，提高稳定性、可维护性和排错能力。

## 已完成修改

1. 新增 `src/logger.js`
   - 自动写入本地日志文件。
   - 日志位置：Electron `userData/logs` 目录。
   - 主进程 IPC 异常、启动退出、生成失败、文件读取失败、归档失败都会记录。

2. 新增 `src/config.js`
   - 自动保存常用配置到 `userData/config.json`。
   - 当前已保存：监控文件夹、模板路径、是否自动监控、网报登录地址等基础配置。
   - 启动时优先使用上次成功监控的文件夹。

3. 修复教育事业年报导入状态不能持久化的问题
   - 导入教育事业年报后，现在会保存到数据库 `edu_report` 表。
   - 软件重启后，如果原 Excel 文件还存在，会自动恢复已导入状态。

4. IPC 统一异常保护
   - 新增 `handleIpc()` 包装函数。
   - 避免主进程某个功能报错时直接造成前端无响应。
   - 失败时统一返回 `{ ok: false, message: "错误信息" }`。

5. 模板路径选择更稳
   - 新增 `getLayoutTemplatePath()`。
   - 查找顺序：配置中的模板路径 → 监控目录下的 `经费年报模板.xlsx` → 默认 `陇集/经费年报模板.xlsx`。

6. 日志界面增强
   - 在“处理日志”标签页增加“打开日志目录”按钮。
   - 方便以后出错时直接找到日志文件。

7. 监控器错误记录
   - Excel 读取失败、归档失败不再静默忽略，会写入日志。

## 2026-05-10 验证码刷新问题修复

### 修复内容
1. 自动登录不再通过 Node.js 单独请求验证码 URL 识别图片。
   - 原因：该网站访问 `getVerifyCode` 可能会重新生成验证码，导致 OCR 识别结果与页面当前显示的验证码不一致。
   - 新逻辑：直接在 webview 页面内用 canvas 截取当前显示的验证码图片，再交给 OCR 识别。

2. 修复自动登录重试时报错：`autoLogin is not defined`。
   - 原代码重试时调用了不存在的 `autoLogin`。
   - 现在改为调用正确的 `doAutoLogin(account, retryCount + 1)`。

3. 日志提示优化：
   - 会显示“已截取当前页面验证码图片”。
   - 会显示页面验证码来源 URL，但不会再重新下载该 URL。

## 2026-05-10 验证码截图二次修复

### 修复问题
- 页面内 `webview.executeJavaScript()` 截取验证码图片时，在目标网站上可能抛出 `GUEST_VIEW_MANAGER_CALL`，导致拿不到验证码。
- 旧逻辑拿不到验证码后仍然提交登录，造成空验证码反复失败。

### 调整内容
- 新增双通道验证码获取：
  1. 先尝试页面内 canvas 读取当前验证码图片；
  2. 失败后改用 Electron `webview.capturePage()` 按验证码图片位置截图裁剪。
- 不再重新下载验证码 URL，避免刷新服务器验证码。
- 如果验证码无法识别，立即停止自动登录，不再空验证码提交。
- 保留重试机制，但仅在已经提交过有效验证码后根据登录结果重试。

### 预期日志
正常情况下应看到：

```text

[自动登录] 页面内取验证码失败: ...，尝试 Electron 截图裁剪
[自动登录] 已获取当前页面验证码图片(...，方式：webview-capturePage)
[自动登录] 正在识别当前页面验证码...

```

## v2.0.4 验证码人工兜底修复

### 本次问题
日志显示 `webview.executeJavaScript()` / `webview.capturePage()` 在该登录页面执行失败，导致程序无法截取当前页面验证码。

### 修复内容
1. 保留自动截图/OCR方案。
2. 当自动截图或OCR失败时，不再空验证码提交。
3. 新增手动验证码兜底：程序会弹出输入框，请用户看右侧网页当前验证码后手动输入。
4. 手动输入后，程序继续自动填写用户名、密码、验证码并点击登录。

### 使用方式
点击账号“登录”后，如果自动识别验证码失败，会弹出提示框。看右侧网页验证码，输入后点“确定”即可继续。
